# 科目管理功能代码解释文档

## 一、功能概述

科目管理模块实现了对教学科目的**增删改查（CRUD）**功能，包括：
- **查询**：按名称模糊搜索科目列表
- **添加**：新增科目（名称、状态）
- **修改**：编辑已有科目的名称和状态
- **删除**：根据ID删除科目

---

## 二、技术架构分层

采用经典的三层架构模式：

```
前端页面 (HTML + JavaScript)
    ↓ AJAX请求
Controller层 (SubjectController.java) — 接收请求、返回响应
    ↓ 调用
Service层 (SubjectService.java / SubjectServiceImpl.java) — 业务逻辑处理
    ↓ 调用
DAO层 (SubjectDao.java / SubjectDaoImpl.java) — 数据库操作
    ↓ SQL执行
数据库 (MySQL)
```

### 3.3 DAO实现 — `dao/SubjectDaoImpl.java`

#### 3.3.1 新增科目

```java
public int addSubject(Subject subject) {
    String sql = "INSERT INTO subject (name, creator, status) VALUES (?, ?, ?)";
    try (Connection conn = DbUtil.getConnection();
         PreparedStatement ps = conn.prepareStatement(sql)) {
        ps.setString(1, subject.getName());
        ps.setString(2, subject.getCreator());
        ps.setString(3, subject.getStatus());
        return ps.executeUpdate(); // 返回影响行数
    } catch (Exception e) {
        e.printStackTrace();
        return 0;
    }
}
```

**关键点**：
- 使用 `PreparedStatement` 防止SQL注入攻击
- `try-with-resources` 自动关闭数据库连接，避免资源泄漏
- `executeUpdate()` 返回受影响的行数，1表示成功

#### 3.3.4 查询全部科目

```java
public List<Subject> findAllSubjects() {
    List<Subject> subjects = new ArrayList<>();
    String sql = "SELECT * FROM subject";
    try (Connection conn = DbUtil.getConnection();
         PreparedStatement ps = conn.prepareStatement(sql);
         ResultSet rs = ps.executeQuery()) {
        while (rs.next()) {
            Subject s = new Subject();
            s.setId(rs.getInt("id"));
            s.setName(rs.getString("name"));
            s.setCreator(rs.getString("creator"));
            s.setStatus(rs.getString("status"));
            subjects.add(s);
        }
        return subjects;
    } catch (Exception e) {
        e.printStackTrace();
        return subjects;
    }
}
```

**关键点**：
- `ResultSet` 遍历查询结果，逐行封装为 `Subject` 对象
- 使用 `try-with-resources` 确保 `ResultSet`、`PreparedStatement`、`Connection` 都被关闭

### 3.5 Service实现 — `Service/SubjectServiceImpl.java`

```java
public class SubjectServiceImpl implements SubjectService {

    private SubjectDao subjectDao = new SubjectDaoImpl();

    @Override
    public boolean addSubject(Subject subject) {
        // 参数校验：科目名不能为空
        if (subject.getName() == null || subject.getName().trim().isEmpty()) {
            return false;
        }
        return subjectDao.addSubject(subject) > 0;
    }

    @Override
    public boolean deleteSubject(Integer id) {
        // 校验id有效性
        if (id == null || id <= 0) {
            return false;
        }
        return subjectDao.deleteSubject(id) > 0;
    }

    @Override
    public boolean updateSubject(Subject subject) {
        // 校验id和科目名
        if (subject.getId() == null || subject.getId() <= 0
                || subject.getName() == null || subject.getName().trim().isEmpty()) {
            return false;
        }
        return subjectDao.updateSubject(subject) > 0;
    }

    @Override
    public List<Subject> findAllSubjects() {
        return subjectDao.findAllSubjects();
    }

    @Override
    public List<Subject> findSubjectsByName(String name) {
        // 若名称为空，查所有
        if (name == null || name.trim().isEmpty()) {
            return subjectDao.findAllSubjects();
        }
        return subjectDao.findSubjectsByName(name.trim());
    }
}
```

**关键点**：
- **参数校验**：在Service层进行业务规则校验（如名称非空、ID有效）
- **空值处理**：查询名称为空时，自动转为查询全部
- **返回值转换**：DAO返回影响行数（int），Service转为布尔值（boolean）

### 3.6 控制层（Controller）— `controller/SubjectController.java`

```java
@RestController
@RequestMapping("/subject")
public class SubjectController {

    private final SubjectService subjectService = new SubjectServiceImpl();

    // 查询科目列表
    @GetMapping("/list")
    public String list(@RequestParam(required = false) String name) {
        try {
            List<Subject> subjects = subjectService.findSubjectsByName(name);
            return JSON.toJSONString(subjects); // 转为JSON返回
        } catch (Exception e) {
            return "{\"error\":\"查询失败，请稍后重试\"}";
        }
    }

    // 添加科目
    @PostMapping("/add")
    public String add(@RequestParam String name,
                      @RequestParam String status,
                      HttpSession session) {
        // 登录校验
        String creator = (String) session.getAttribute("username");
        if (creator == null || creator.isEmpty()) {
            return "{\"error\":\"请先登录后再操作\"}";
        }
        try {
            Subject subject = new Subject();
            subject.setName(name);
            subject.setCreator(creator);
            subject.setStatus(status);
            boolean success = subjectService.addSubject(subject);
            if (success) {
                return "{\"success\":\"添加成功\"}";
            }
            return "{\"error\":\"添加失败\"}";
        } catch (Exception e) {
            return "{\"error\":\"添加失败，请稍后重试\"}";
        }
    }

    // 更新科目
    @PostMapping("/update")
    public String update(@RequestParam Integer id,
                         @RequestParam String name,
                         @RequestParam String status) {
        try {
            Subject subject = new Subject();
            subject.setId(id);
            subject.setName(name);
            subject.setStatus(status);
            boolean success = subjectService.updateSubject(subject);
            if (success) {
                return "{\"success\":\"更新成功\"}";
            }
            return "{\"error\":\"更新失败\"}";
        } catch (Exception e) {
            return "{\"error\":\"更新失败，请稍后重试\"}";
        }
    }

    // 删除科目
    @GetMapping("/delete")
    public String delete(@RequestParam Integer id) {
        try {
            boolean success = subjectService.deleteSubject(id);
            if (success) {
                return "{\"success\":\"删除成功\"}";
            }
            return "{\"error\":\"未找到该科目\"}";
        } catch (Exception e) {
            return "{\"error\":\"删除失败，请稍后重试\"}";
        }
    }
}
```

**关键点**：
- `@RestController`：标识为RESTful控制器，返回JSON数据
- `@RequestMapping("/subject")`：基础路径前缀
- `@GetMapping("/list")`：处理GET请求，查询数据
- `@PostMapping("/add")`：处理POST请求，添加数据
- `@RequestParam`：从请求参数中获取值
- `HttpSession`：获取当前登录用户，用于鉴权
- `JSON.toJSONString()`：将Java对象序列化为JSON字符串

#### 3.7.2 AJAX查询科目列表

```javascript
function loadSubjects(name = "") {
    $.ajax({
        url: "/sft/subject/list",   // 请求后端接口
        type: "GET",                // GET请求
        data: { name: name },       // 传递查询参数
        dataType: "json",           // 期望返回JSON
        success: function (subjects) {
            let tbody = document.getElementById("subjectTableBody");
            tbody.innerHTML = "";   // 清空原有内容
            subjects.forEach(subject => {
                let tr = document.createElement("tr");
                tr.innerHTML = `
                    <td>${subject.id}</td>
                    <td>${subject.name}</td>
                    <td>${subject.creator}</td>
                    <td>${subject.status}</td>
                    <td>
                        <a href="#" onclick="deleteSubject(${subject.id})">删除</a>
                        <a href="subjectUpdate.html?id=${subject.id}">修改</a>
                    </td>
                `;
                tbody.appendChild(tr);
            });
        },
        error: function () {
            alert("加载科目失败，请重试");
        }
    });
}
```

**关键点**：
- 使用 jQuery 的 `$.ajax` 发送异步请求
- `dataType: "json"` 自动将返回的JSON字符串解析为JavaScript对象
- 动态生成表格行，实现无刷新页面更新

## 四、数据库表结构

```sql
CREATE TABLE `subject` (
    `id` INT AUTO_INCREMENT PRIMARY KEY,        -- 科目ID（自增主键）
    `name` VARCHAR(100) NOT NULL,               -- 科目名称
    `creator` VARCHAR(50) DEFAULT NULL,         -- 创建人
    `status` VARCHAR(10) DEFAULT '正常',         -- 状态（正常/禁用）
    `create_time` TIMESTAMP DEFAULT CURRENT_TIMESTAMP  -- 创建时间
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4;
```

## 五、总结

科目管理模块完整实现了MVC三层架构：

| 层级 | 文件 | 职责 |
|------|------|------|
| 表现层 | `subjectList.html` / `subjectAdd.html` / `subjectUpdate.html` | 用户界面、交互逻辑 |
| 控制层 | `SubjectController.java` | 接收HTTP请求、调用Service、返回JSON |
| 业务层 | `SubjectService.java` / `SubjectServiceImpl.java` | 业务规则校验、事务控制 |
| 数据层 | `SubjectDao.java` / `SubjectDaoImpl.java` | 执行SQL、操作数据库 |
| 实体层 | `Subject.java` | 数据封装、跨层传递 |

这种分层设计使得代码结构清晰、职责单一、便于维护和扩展。
